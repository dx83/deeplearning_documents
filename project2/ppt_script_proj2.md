# 외형 분류 개선 사항
### 1장
외형 분류의 경우, 1차 때 방식은 데이터가 많은 9개 클래스만 분류하고 나머지는 기타 처리하는 방식이었음  
이번 2차에서는 29개를 모두 분류합니다.

검색 과정  
    이미지 입력 =>  
    알약 영역 분리 (이미지 설명) =>  
    알약 모양을 숫자로 이루어진 벡터로 변환 =>  
    각 외형의 대표 벡터와 거리 비교 =>  
    가장 가까운 외형 후보를 반환  

### 2장
56차원 형상 임베딩 다음으로 구성  
각 차원은 해당 숫자 개수로 이루어진 벡터  
    기본 기하 특징 : 얼마나 길쭉하고, 둥글고, 오목한가를 벡터로 표현  
    EFD : 외곽선 좌표 벡터로 전환  
    방사형 FFT : 중심에서 외곽까지 거리를 벡터로 표현  
    Hu moment : 면적 분포의 패턴을 벡터화  

### 3장
실제 이미지에서 확인한 외형 검색 결과는  
보시는바와 같이 장방형, 사과형, 땅콩형  
1등 후보가 일치하는 모습을 보여줌

<br>

# 각인 인식 개선 사항
### 1장
1차에서 사용한 CRNN에서 다음과 같은 문제 발생  
    2줄 각인 => 일부만 해석  
    각인이 가득찬 경우 => 양 끝의 문자를 틀리게 해석  
    숫자,알파벳 혼동  

### 2장
기존 CRNN 에서 CNN과 Transformer를 결합한 방식을 2차에서 사용함  

주요 강점 => 1줄 순차 인식 | 전체 문맥 파악  
공간 정보 처리 => 1차원적 처리 | 행과열의 2차원 처리  
어려운 배치 => 설명안 함  
연산량 => 비교적 적은 | 연산량 증가  
이 프로젝트는 실제 알약의 불규칙한 각인 배치를 더 잘 찾아내기 위해 이 비용을 감수하기로 했습니다.  

### 3장
2차 방식으로 인식 개선 : 예시 => CRNN 오인식 => 개선 정답  
    크롭이 제대로 되지 않고 2줄 각인  
    각인 불규칙 배치
    꽉찬 각인으로 잘린 부분  
    문자 혼동

<br>

# 챗봇 개선 사항
### 1장
기존의 문제 : 각인을 읽을 수 없다면 => 챗봇상 "못 찾음"

챗봇 흐름을 보시면
1. 각인이 모호 | 소실 => 사용자가 각인 없다고 답하면
2. 추가적인 질문을 요구 : 색상, 모양, 분할선 유무
3. 개선 사항 : 무슨 증상에 먹는 약인지 => 복용목적도 입력 받음

### 2장
증상 입력  
LLM 매칭  
최종 후보군 나열  

사용자가 표현하는 복용 목적은 개인마다 다를 수 있어서, 단순 키워드나 규칙만으로는 의약품 효능과 정확히 연결하기 어렵습니다. LLM을 활용하면 다양한 자연어 표현의 의미와 맥락을 이해해서 복용 목적에 적합한 후보 알약을 찾아낼 수 있습니다.
