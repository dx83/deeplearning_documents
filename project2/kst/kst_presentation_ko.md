# 슬라이드 1 — 알약 각인 OCR에 CNN + Transformer를 사용하는 이유

## 목적

알약 각인이 항상 깔끔하고 일정한 간격의 한 줄 텍스트로 배치되지 않는 상황에서, 기존 CRNN을 넘어 새로운 구조를 선택한 이유를 설명합니다.

## 슬라이드 내용

### 순차적 읽기에서 공간적 이해로

알약 각인에는 넓은 문자 간격, 곡선형 배열, 불규칙한 문자 위치가 나타날 수 있습니다. CNN + Transformer는 2차원 배치를 유지하면서 각인 영역 전체의 시각적 정보를 서로 연결할 수 있습니다.

| | CRNN | CNN + Transformer |
|---|---|---|
| 주요 강점 | 규칙적인 한 줄 텍스트를 효율적으로 인식 | 각인 영역 전체의 전역 문맥 파악 |
| 공간 정보 처리 | 일반적으로 특징 맵을 1차원 시퀀스로 변환 | 2D 위치 인코딩으로 행과 열 정보 유지 |
| 어려운 배치 | 문자 간격이 넓거나 정렬이 불규칙하면 대응이 제한적 | 어텐션으로 멀리 떨어지거나 불규칙한 위치의 문자를 연결 |
| 연산량 | 비교적 적음 | 시각 토큰 수가 증가할수록 연산량 증가 |

**핵심 메시지:** 실제 알약에서 나타나는 불규칙한 공간 배치를 더 잘 표현하기 위해 더 높은 연산 비용을 감수합니다.

## 권장 시각 자료

- 왼쪽: `CNN 특징 → 왼쪽에서 오른쪽으로 이어지는 시퀀스 → 순환 신경망 인식`으로 구성된 간단한 CRNN 흐름도
- 가운데: 문자 간격이 넓은 알약과 문자 위치가 불규칙한 알약 예시
- 오른쪽: `CNN 특징 → 2D 위치 정보 → 전역 어텐션 → 각인 디코딩`으로 구성된 프로젝트 모델 흐름도
- 멀리 떨어진 문자 사이에 어텐션 연결선을 표시하여 수식 없이도 “전역 문맥”을 이해할 수 있도록 구성

## 발표 가이드 (~30초)

CRNN은 효율적이며 규칙적인 한 줄 텍스트를 인식하는 데 여전히 적합한 모델입니다. 하지만 알약 각인의 배치는 예측하기 어렵습니다. 문자 사이의 간격이 매우 넓거나 서로 다른 높이에 배치될 수 있습니다. 먼저 CNN이 모서리, 홈, 획과 같은 국소 특징을 추출합니다. 그런 다음 Transformer가 전체 영역의 관계를 파악하기 전에 2차원 위치 정보를 추가합니다. 이 구조는 불규칙한 배치를 표현하는 데 더 적합하지만 연산량이 증가한다는 단점이 있습니다. 이는 모델 구조를 선택한 근거이며, 정확도가 더 높다고 입증된 결과는 아닙니다. 이를 입증하려면 동일한 조건에서 CRNN과 비교하는 실험이 필요합니다.

# 슬라이드 2 — 더 현실적인 데이터셋을 위한 부트스트래핑

## 목적

이전 CRNN 프로젝트가 데이터 준비 과정을 어떻게 가속했는지, 그리고 실제 알약 이미지가 합성 학습 데이터를 어떻게 보완하는지 설명합니다.

## 슬라이드 내용

### 이전 모델을 재사용하고 데이터 품질을 개선

`기존 각인 검출 모델 → 초기 영역 라벨 생성 → 사람의 검토 및 수정 → OCR 학습용 이미지 영역`

| 데이터셋 구성 | 크기 | 역할 |
|---|---:|---|
| 필터링된 Synth90k (A–Z, 0–9) | 28k | 다양한 문자 조합과 정제된 합성 이미지 제공 |
| 라벨링된 실제 알약 이미지 | 5k | 실제 각인, 조명, 질감, 초점 및 위치 변화 반영 |

**핵심 메시지:** 이전 검출 모델로 라벨링 작업을 줄이고, 사람의 검토와 실제 이미지 추가를 통해 OCR 데이터셋을 실제 알약 인식 환경에 더 적합하게 개선했습니다.

## 권장 시각 자료

- 정보가 많은 표만 사용하기보다 왼쪽에서 오른쪽으로 이어지는 3단계 파이프라인으로 구성
- 1단계: 기존 검출 모델이 알약 이미지에서 각인 영역을 제안
- 2단계: 검토자가 제안된 영역을 승인하거나 수정
- 3단계: 검증된 이미지 영역을 합성 및 실제 학습 데이터셋에 추가
- 마지막에 `합성 이미지 28k + 실제 이미지 5k`를 간결한 데이터셋 그래픽으로 표시

## 발표 가이드 (~30초)

새로운 데이터셋 구축을 처음부터 시작하지는 않았습니다. 이전 CRNN 기반 프로젝트에서 개발한 각인 영역 검출 모델을 활용해 실제 알약 이미지의 초기 영역 라벨을 생성했습니다. 이후 사람이 해당 영역을 검토하고 수정한 뒤 OCR 학습용 이미지로 사용했습니다. 필터링된 합성 이미지 28,000장은 영문 대문자와 숫자의 조합을 안정적으로 제공합니다. 실제 알약 이미지 5,000장은 합성 텍스트만으로 충분히 재현하기 어려운 각인 깊이, 빛 반사, 알약 질감, 흐림, 불규칙한 위치와 같은 조건을 추가합니다. 따라서 핵심 개선점은 단순히 데이터 수가 늘어난 것이 아니라, 학습 데이터가 실제 사용 환경에 더 가까워졌다는 점입니다.

# 슬라이드 3 — 네이티브 Android 앱 구성 계획

## 목적

모바일 앱이 이미지 촬영 품질을 개선하고 가벼운 작업을 기기에서 처리하는 한편, 연산량이 많은 OCR 모델은 백엔드에서 실행하는 구조를 설명합니다.

## 슬라이드 내용

### 촬영과 경량 처리는 엣지에서, 고연산 인식은 백엔드에서

`Android 카메라 → 기기 내 이미지 자르기 및 검사 → 백엔드 CNN + Transformer OCR → 인식 결과`

**Android 기기에서 처리**

- 초점과 노출을 제어하여 각인을 더 선명하게 촬영
- 업로드 전에 알약 영역을 찾고 필요한 부분만 자르기
- 간단한 챗봇 감정 분석과 같은 경량 기능을 짧은 지연 시간으로 실행

**백엔드에서 처리**

- 연산량이 많은 CNN + Transformer OCR 모델 실행
- 인식된 각인 정보를 앱으로 반환
- 앱을 새로 배포하지 않고도 인식 모델을 중앙에서 업데이트

**핵심 메시지:** 작업의 특성에 따라 처리 위치를 분리합니다. 빠른 응답과 카메라 연동이 필요한 작업은 스마트폰에서 처리하고, 연산량이 많은 인식은 서버에서 수행합니다.

## 권장 시각 자료

- 왼쪽에는 스마트폰 영역, 오른쪽에는 클라우드/백엔드 영역을 구분하여 표시
- 스마트폰 내부에는 `카메라 제어 → 알약 영역 자르기 → 이미지 품질 검사` 흐름을 배치
- 전처리된 이미지 영역만 백엔드로 전송되고, 짧은 인식 결과가 스마트폰으로 돌아오는 흐름을 표현
- 경량 챗봇 분석은 이미지 처리 흐름 옆에 배치하여 OCR의 일부가 아닌 별도의 기기 내 기능임을 명확히 표시
- 전체 다이어그램에 **구성 계획**이라고 표기하여 이미 운영 중인 완성 기능으로 오해하지 않도록 구성

## 발표 가이드 (~30초)

네이티브 앱은 인식 과정의 프런트엔드로 개발할 계획입니다. Android 카메라 기능을 활용하면 초점과 노출을 직접 제어할 수 있으므로, 흐리거나 빛을 반사하는 각인을 인식 전에 더 선명하게 촬영할 수 있습니다. 알약 영역 자르기, 기본적인 이미지 품질 검사, 간단한 챗봇 감정 분석처럼 빠른 응답이 필요한 경량 작업은 기기에서 처리합니다. 이후 전처리된 알약 이미지를 백엔드로 전송하고, 연산 비용이 높은 CNN + Transformer 모델로 인식합니다. 이러한 구조는 모바일 앱의 반응성을 유지하면서 데이터셋과 모델이 개선될 때 OCR 모델을 중앙에서 업데이트할 수 있게 합니다.
