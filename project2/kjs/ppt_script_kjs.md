# 1장 챗봇
기존의 문제 : 각인을 읽을 수 없다면 => 챗봇상 "못 찾음"

챗봇 흐름상
1. 각인이 모호 | 소실 => 사용자가 각인 없다고 답하면
2. 추가적인 질문을 요구 : 색상, 모양, 분할선 유무
개선
3. 무슨 증상에 먹는 약인지 => 복용목적도 입력 받음


# 2장 챗봇
증상 입력
LLM 매칭
최종 후보군 나열

사용자가 표현하는 복용 목적은 개인마다 다를 수 있어서, 단순 키워드나 규칙만으로는 의약품 효능과 정확히 연결하기 어렵습니다. LLM을 활용하면 다양한 자연어 표현의 의미와 맥락을 이해해서 복용 목적에 적합한 후보 알약을 찾아낼 수 있습니다.
