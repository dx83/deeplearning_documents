pill_shape_centroids.pkl은 분류 모델 자체가 아니라, unknown 판정을 위한 보조 인덱스 파일입니다.

  쉽게 말하면:

  > 각 알약 모양 클래스의 “대표 임베딩 벡터”와 “유사도 통과 기준값”을 저장한 파일입니다.

  생성 내용은 대략 이렇습니다.

   항목                     의미
  ━━━━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   class_names              9개 클래스 이름 순서
  ───────────────────────  ─────────────────────────────────────────────────────────────
   centroids                클래스별 대표 embedding 벡터, shape (9, 1280)
  ───────────────────────  ─────────────────────────────────────────────────────────────
   thresholds               클래스별 유사도 임계값
  ───────────────────────  ─────────────────────────────────────────────────────────────
   statistics               클래스별 기준 이미지 수, 평균 유사도, 최소 유사도 등
  ───────────────────────  ─────────────────────────────────────────────────────────────
   image_count              centroid 생성에 사용한 기준 이미지 수
  ───────────────────────  ─────────────────────────────────────────────────────────────
   model_path               기준 모델 경로
  ───────────────────────  ─────────────────────────────────────────────────────────────
  2. 모델 중간의 GlobalAveragePooling2D 출력을 embedding 모델로 사용
  3. 학습/reference 이미지들을 넣어서 1280차원 embedding 추출
  4. 같은 클래스 이미지들의 embedding 평균을 내서 클래스별 centroid 생성
  5. 클래스별로 “이 정도 유사도 이상이면 같은 클래스라고 볼 수 있다”는 threshold 계산
  6. 그 결과를 pill_shape_centroids.pkl로 저장

  예측할 때는 이렇게 씁니다.

  - 모델 softmax 결과: “분류상 가장 가능성 높은 클래스”
  - centroid similarity 결과: “embedding상 가장 가까운 클래스”
  - 둘이 같은 클래스이고
  - softmax 확률이 0.60 이상이고
  - similarity가 해당 클래스 threshold 이상이면

  known으로 인정합니다. 아니면 unknown 처리합니다.

  즉 pill_shape_centroids.pkl의 목적은 등록된 클래스와 비슷한지 확인해서 미등록/애매한 이미지를 걸러내는 것입니다. 분류
  모델의 약점을 보완하는 유사도 기반 검증 파일이라고 보면 됩니다.
